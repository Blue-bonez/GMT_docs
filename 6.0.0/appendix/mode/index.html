16.4 GMT现代模式¶

自GMT6起，GMT引入了一种全新的命令执行模式，称之为现代模式。GMT5以及之前的命令风格则称之为经典模式。

注解

GMT6既支持经典模式也支持现代模式，因而GMT6可以完全兼容GMT5的代码。

从上面的示例脚本中以及实际使用经验中会发现GMT经典模式存在的一些明显的缺点或问题：

  1. 大部分命令需要使用 -K 和 -O 选项，而且极其容易出错
  2. 需要使用 -P 选项决定画布的摆放方式
  3. 每个绘图命令都需要使用重定向符号 > 或 >>
  4. 每个绘图命令中都需要给定PS文件的名字
  5. 大部分命令都需要重复使用 -J 和 -R 选项，尽管具体数值可以省略
  6. 需要调用单独的命令将生成的PS文件转换为PDF、JPG、PNG等格式

现代模式下的GMT命令风格具有如下优点：

  1. 不再需要考虑 -K、-O 和 -P 选项
  2. 不再需要进行重定向，也无需为每个命令都指定PS文件名
  3. 不再需要为每个命令都是用 -R 和 -J 选项
  4. 最终可生成PDF、PNG等多种格式的图片文件，无需再转换
  5. 自动确定纸张大小并进行裁剪
  6. 整个脚本在独立的临时文件夹下执行，因而可以同时执行多个脚本而不互相干扰

16.4.3 从经典到现代¶

将一个经典模式的脚本改成现代模式下的脚本基本遵循如下几点：

  1. 绘图脚本以 gmt begin figname pdf 开头，以 gmt end 结束
  2. 去掉 -K、-O、-P、重定向符号以及PS文件名
  3. 去掉多余的 -R 和 -J 选项
  4. 以 ps 开头的模块改名了，比如 pscoast => coast、psbasemap => basemap。几个例外包括 psxy => plot，psxyz => plot3d，psscale => colorbar。

现代模式下提供了几个新的功能（详细用法见模块手册）：

  • gmt begin 开始现代绘图模式
  • gmt end 结束现代绘图模式
  • gmt figure 设置当前图片的基本信息
  • gmt subplot 子图模式及相关设置

16.4.4 现代模式: 子图模式¶
